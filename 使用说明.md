# py_mcp_full 使用说明

> py_mcp_full 是 py_mcp v3.0 的全量增强版，在原有 10 个影视爬虫开发工具基础上新增 8 个文件操作工具，支持完整的文件管理 + 爬虫源开发工作流。

## 版本信息

- **版本**: 3.1.0
- **服务名**: py-mcp-full
- **协议**: HTTP Stream (MCP)
- **运行环境**: Node.js
- **工作根目录**: `/storage/emulated/0/lz/AAA/py/`（通过 `ROOT` 环境变量配置）

## 外部MCP客户端导入配置

### Claude Desktop / Cursor / Cherry Studio 等标准客户端

粘贴以下JSON到 MCP 配置中（args 需改为你的实际绝对路径）：

```json
{
  "mcpServers": {
    "py_mcp_full": {
      "command": "node",
      "args": ["/storage/emulated/0/Download/Operit/mcp_plugins/py_mcp_full/index.js"],
      "env": {
        "ROOT": "/storage/emulated/0/lz/AAA/py/"
      }
    }
  }
}
```

> 注意：`args` 中必须使用 `index.js` 的绝对路径。`ROOT` 按实际爬虫源目录修改。首次使用前需在 `py_mcp_full` 目录下执行 `npm install` 安装依赖。

## 工具总览（18个）

### 一、爬虫开发工具（原有10个）

| # | 工具名 | 功能 | 关键参数 |
|---|--------|------|----------|
| 1 | `analyze_website` | 分析网站结构，识别CMS类型 | `url` |
| 2 | `create_spider_source` | 生成爬虫源代码 | `url`, `cms_type`, `mode` |
| 3 | `debug_selector` | 调试XPath/CSS选择器 | `url`, `selector`, `selector_type` |
| 4 | `debug_play_link` | 递归调试播放链接提取 | `url`, `flag` |
| 5 | `test_interface` | 测试爬虫五大接口 | `source_code`, `test_type` |
| 6 | `evaluate_source` | 评估爬虫源规范符合度 | `source_code` |
| 7 | `fetch_url` | 抓取URL内容 | `url`, `method`, `headers` |
| 8 | `edit_file` | 编辑已有文件 | `path`, `old`, `new` |
| 9 | `find_in_file` | 文件内容搜索 | `path`, `pattern`, `regex` |
| 10 | `list_directory` | 列出目录内容 | `path` |

### 二、文件操作工具（新增8个）

| # | 工具名 | 功能 | 参数 | 说明 |
|---|--------|------|------|------|
| 11 | `read_file` | 读取文件内容 | `path`, `offset`(行号偏移), `limit`(行数) | 支持分段读取大文件 |
| 12 | `write_file` | 写入文件 | `path`, `content`, `append`(是否追加) | append=true 追加，false 覆盖 |
| 13 | `delete_file` | 删除文件/目录 | `path`, `recursive`(递归删除) | 删除目录需 recursive=true |
| 14 | `create_directory` | 创建目录 | `path`, `recursive`(递归创建) | 类似 mkdir -p |
| 15 | `move_file` | 移动/重命名文件 | `source`, `destination` | |
| 16 | `copy_file` | 复制文件 | `source`, `destination` | |
| 17 | `file_info` | 获取文件信息 | `path` | 大小、修改时间、类型等 |
| 18 | `get_cwd` | 获取当前工作目录 | 无 | 返回 ROOT 环境变量路径 |

## 使用示例

### 1. 爬虫开发流程

```
① analyze_website(url="https://example.com")
   → 识别CMS类型，获取HTML结构参数

② debug_selector(url="...", selector="//div[contains(@class,'item')]", selector_type="xpath")
   → 验证选择器是否正确

③ create_spider_source(url="...", cms_type="macv8", mode="t4")
   → 生成完整爬虫源代码

④ test_interface(source_code="...", test_type="all")
   → 测试五大接口是否正常

⑤ evaluate_source(source_code="...")
   → 检查规范符合度
```

### 2. 文件操作流程

```
① 读取爬虫源文件
   read_file(path="my_spider.py")

② 分段读取大文件（从第50行开始，读30行）
   read_file(path="my_spider.py", offset=50, limit=30)

③ 编辑已有文件（替换选择器）
   edit_file(path="my_spider.py", old='@class="item"', new='contains(@class,"item")')

④ 写入新文件
   write_file(path="new_spider.py", content="...")

⑤ 追加内容到文件末尾
   write_file(path="log.txt", content="新日志行\n", append=true)

⑥ 搜索文件内容
   find_in_file(path="my_spider.py", pattern="vod_play", regex=true)

⑦ 复制备份
   copy_file(source="my_spider.py", destination="my_spider.py.bak")

⑧ 移动/重命名
   move_file(source="draft.py", destination="production.py")

⑨ 查看文件信息
   file_info(path="my_spider.py")

⑩ 获取工作目录
   get_cwd() → "/storage/emulated/0/lz/AAA/py/"
```

## 常见问题

**Q: 启动后工具不可用？**
A: 检查 `index.js` 所在目录是否执行过 `npm install`（如依赖外部包）。

**Q: 文件路径报错？**
A: 确认 `ROOT` 环境变量指向正确目录，或使用绝对路径。

**Q: 与 py_mcp 的关系？**
A: py_mcp_full 是 py_mcp 的超集，完全兼容且多了8个文件操作工具。建议直接使用 py_mcp_full。

**Q: 如何只用文件操作工具？**
A: 可单独启用 `py_mcp_full` 并禁用 `py_mcp`，功能完全覆盖。
